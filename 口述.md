## 口述正文

### 起点：用户点击按钮

用户在 EditorPage 的 Agent 面板输入创作指令，点击「运行 Agent」按钮，触发 `runAgent` 函数。

函数先做两件事：第一，从 Zustand store 取 LLM 配置，API Key 为空则提醒用户。第二，用防抖 timer 把编辑器已有内容自动保存到后端。

确认 Key 有效后，调用 `api.agent.runAgentStream()`。用 fetch 发起 POST 到 `/api/novels/{id}/agent/run`，body 带 section_id、api_key、model、instruction 等参数。

为什么不用 EventSource？因为 EventSource 只支持 GET，而我们必须 POST 传 API Key。

请求发出去后，前端拿到 `res.body`，调用 `getReader()` 获取 ReadableStream 读取器，进入 `while(true)` 循环。每次 `await reader.read()` 拿到字节数据，TextDecoder 解码后追加到 buffer。buffer 按换行分割，匹配 `data:` 前缀的行，JSON.parse 解析后调用 `onEvent` 回调更新 React UI。

### 后端：LangChain Agent

请求到达 FastAPI，路由 `run_agent` 接收。Pydantic 校验参数后，用用户配置的 api_key、base_url、model 实例化 `NovelAgent`。

NovelAgent 内部：
1. 用 `ChatOpenAI` 创建 LLM（`langchain-openai`）
2. 用 `get_all_tools()` 注册 10 个工具
3. 用 `create_react_agent` 创建 LangGraph Agent，传入 AGENT_SYSTEM_PROMPT

### Agent 执行流程

调用 `agent.astream()` 后，用 `astream_events` 监听事件流，通过 `StreamingResponse` 持续推送 SSE。

```
用户指令 → LLM思考 → 工具调用 → 观察结果 → LLM思考 → ... → finish
```

执行过程监听的事件：

1. **on_chat_model_start** → 推送 "thinking" 事件
2. **on_tool_start** → 推送 "tool_call" 事件，包含工具名和输入
3. **on_tool_end** → 推送 "tool_call" done 事件
4. **on_chat_model_end** → 推送 "thinking" done 事件

Agent 决定调用 finish 工具时，ReAct 循环结束，返回最终正文。

### 10 个工具

| 工具 | 做什么 |
|------|--------|
| `get_outline` | 遍历大纲树，输出卷→章→节完整结构 |
| `get_characters` | 读取角色卡片，包括外貌/性格/背景/关系等 |
| `get_world_settings` | 读取世界观设定，按类别输出 |
| `get_summaries` | 读取最近 5 个已完成节的摘要 |
| `search_memory` | 用 sentence-transformers 编码查询，在 ChromaDB 中余弦相似度搜索 |
| `write_content` | 占位工具，实际写作由 LLM 在 ReAct 循环中完成 |
| `polish_content` | 同上，占位 |
| `review_content` | 同上，占位 |
| `summarize_content` | 同上，占位 |
| `finish` | 返回最终结果，结束循环 |

Note: write/polish/review/summarize 工具的"实际干活"发生在 LLM 在 ReAct 循环中的推理阶段。工具本身是 placeholder，真正的写作逻辑由 LLM 直接生成并通过 finish 返回。

### 保存和 RAG 嵌入

Agent 完成后，后端拿到 `final_content`：
1. `storage.save_section_content()` 写入 `sections/{sectionId}.txt`
2. 内容 > 50 字时调用 `memory.embed_section()`，用 sentence-transformers 编码后 upsert 到 ChromaDB
3. 下次 `search_memory` 工具就能检索到本节

### 摘要生成

生成摘要是独立端点 `/api/novels/{id}/agent/summrize`：
1. 用 `ChatOpenAI` 直接调用，skill 来自 `skills/summary.md`
2. 解析 JSON 得到 `{summary, key_events, character_state_changes, world_setting_changes}`
3. 更新大纲节点的 summary 和 status
4. 追加到 summaries.json

### 数据持久化

JSON 文件存储在 `backend/data/{novel_id}/`：
```
├─ meta.json        # 项目元信息
├─ outline.json     # 大纲树
├─ characters.json  # 角色卡片
├─ world_settings.json  # 世界观设定
├─ summaries.json   # 摘要
└─ sections/        # 各节正文 .txt
```
